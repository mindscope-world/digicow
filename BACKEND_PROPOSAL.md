# Backend Development Proposal: DigiCow Africa Ltd AI-Powered Farmer Intelligence System

## Executive Summary

This proposal outlines the backend architecture for DigiCow Africa Ltd's AI-powered farmer intelligence system using FastAPI and Neo4j. The backend will power a data-driven extension service platform targeting youth extension agents in Kenya, enabling them to deliver personalized, data-driven advisory services that improve farmer productivity and input adoption.

The system will transform the existing farmer data (currently in CSV format) into a rich knowledge graph that connects farmer behaviors, environmental conditions, input supply chains, and advisory outcomes - enabling extension agents to understand farmer needs and decide what support comes next.

## Current State Analysis

### Frontend Analysis
- Built with TanStack Start (React) + TypeScript + Vite
- Existing farmer data displayed in tabular format with metrics (scores, status, trends)
- Routes include farmers overview, individual farmer details, clusters, outreach, risk, and settings
- Data sourced from `/src/lib/farmers.ts` (hardcoded mock data) and potentially from `backend/Prior_digicow.csv`

### Data Analysis (Prior_digicow.csv)
The CSV contains 100 farmer records with fields including:
- **Farmer ID**: Unique identifier (DC00001-DC00100)
- **Demographics**: Gender, age brackets (18-25, 26-35, 36-45, 46-55, 56-65, 65+)
- **Registration Method**: mobile_app, field_agent, sms, cooperative_office, ussd
- **Cooperative Membership**: Boolean (belong_to_cooperative)
- **Location Hierarchy**: County → Subcounty → Ward
- **Trainer Information**: Array of trainer names
- **Training Topics**: Array of topics (silage_making, milk_hygiene, calf_rearing, breeding_practices, pasture_management, mastitis_control, record_keeping, water_quality, fodder_conservation, AI_insemination, animal_health, feed_management, teat_dipping)
- **Training Status**: has_topic_trained_on (boolean)
- **Training Date**: first_training_date
- **Adoption Metrics**: adopted_within_07_days, adopted_within_90_days, adopted_within_120_days (0/1 flags)

### Core Domain Models (Neo4j Node Labels)

1. **Farmer**
   - Properties: id (string), gender, age_bracket, registration_method, phone, etc.
   - Relationships: BELONGS_TO_COOPERATIVE, LOCATED_IN, TRAINED_BY, PARTICIPATED_IN, HAS_ADOPTED

2. **Cooperative**
   - Properties: name, location_details
   - Relationships: HAS_MEMBER

3. **Location Hierarchy**
   - County → Subcounty → Ward (Hierarchical structure)
   - Properties: name, code (if applicable)

4. **Trainer**
   - Properties: name, specialty_areas, contact_info
   - Relationships: TRAINS_FARMER, SPECIALIZES_IN

5. **TrainingTopic**
   - Properties: name, category, description
   - Relationships: PART_OF_TRAINING, PREREQUISITE_FOR

6. **TrainingSession**
   - Properties: date, location, duration, attendance_count
   - Relationships: HAS_TOPIC, CONDUCTED_BY, ATTENDED_BY

7. **InputProduct** (Seeds, Fertilizers, Animal Health Products)
   - Properties: name, type, brand, cost, effectiveness_rating
   - Relationships: SUITABLE_FOR_TOPIC, PROMOTED_BY, ADOPTED_BY

8. **WeatherStation** (Integrated with Kenya Meteorological Department)
   - Properties: location, last_updated
   - Relationships: MONITORS_WARD, PROVIDES_FORECAST_TO

9. **AdvisoryRecommendation**
   - Properties: recommendation_type, priority_score, confidence_score, generated_at, valid_until
   - Relationships: RECOMMENDED_FOR, BASED_ON_CONDITIONS, SUGGESTS_INPUT

### API Endpoints Design

#### Farmer Management
- `GET /farmers` - List farmers with filtering, pagination, and scoring
- `GET /farmers/{farmer_id}` - Get detailed farmer profile with relationships
- `POST /farmers` - Register new farmer
- `PUT /farmers/{farmer_id}` - Update farmer information
- `DELETE /farmers/{farmer_id}` - Deactivate farmer record

#### Training & Adoption
- `GET /trainings` - List training sessions with filters
- `GET /trainings/{session_id}` - Get training session details
- `POST /trainings` - Record new training session
- `GET /farmers/{farmer_id}/adoption-history` - Get adoption timeline
- `POST /adoptions` - Record new input adoption

#### Advisory & Recommendations (AI-Powered)
- `GET /recommendations/farmer/{farmer_id}` - Get personalized recommendations
- `GET /recommendations/ward/{ward_id}` - Get ward-level recommendations
- `POST /recommendations/generate` - Trigger recommendation generation
- `GET /recommendations/trending-topics` - Get trending training topics by region

#### Input Supply Chain
- `GET /inputs` - List available input products
- `GET /inputs/{product_id}` - Get product details
- `GET /demand-forecast/{ward}` - Get input demand forecast for ward
- `POST /input-requests` - Submit input request from extension agent

#### Analytics & Reporting
- `GET /analytics/adoption-rates` - Get adoption rates by product/topic
- `GET /analytics/trainer-effectiveness` - Get trainer performance metrics
- `GET /analytics/ward-performance` - Get performance metrics by ward
- `GET /analytics/trends` - Get temporal trends in adoption and training

#### Extension Agent Support
- `GET /agents/{agent_id}/dashboard` - Get agent dashboard data
- `GET /agents/{agent_id}/farmers-needing-attention` - Get prioritized farmer list
- `POST /visit-logs` - Log extension agent visit
- `GET /visit-history/{farmer_id}` - Get visit history for farmer

### AI/ML Components

#### Recommendation Engine
1. **Collaborative Filtering**: Recommend trainings/inputs based on similar farmers
2. **Content-Based Filtering**: Recommend based on farmer's profile, location, enterprise type
3. **Contextual Bandits**: For sequential recommendation of visit priorities
4. **Temporal Prediction**: Predict optimal timing for follow-ups and input needs

#### Key Recommendation Types
- **Next Best Training**: Which training topic should this farmer receive next?
- **Input Recommendation**: Which input product would most benefit this farmer?
- **Visit Priority**: Which farmers need attention most urgently?
- **Timing Recommendation**: When should the next visit/input distribution occur?
- **Risk Assessment**: Which farmers are at risk of disengagement or failed adoption?

#### Data Sources for AI
1. Historical training and adoption data
2. Farmer demographics and farm characteristics
3. Location-based factors (weather, soil data - to be integrated)
4. Trainer effectiveness metrics
5. Temporal patterns (seasonality, growth cycles)
6. Social network effects (cooperative membership, trainer networks)

### Data Flow & ETL Pipeline

1. **Data Ingestion**
   - Batch import from existing CSV files
   - Real-time ingestion from frontend forms/APIs
   - Scheduled imports from external sources (weather, market prices)

2. **Data Processing**
   - Data cleaning and normalization
   - Relationship extraction and graph construction
   - Feature engineering for ML models
   - Temporal aggregation (7-day, 30-day, 90-day windows)

3. **Model Training & Inference**
   - Scheduled model retraining (daily/weekly)

4. **API Serving**
   - RESTful endpoints for frontend consumption

   - WebSocket subscriptions for real-time updates
